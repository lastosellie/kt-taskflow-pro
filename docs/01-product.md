# 01. Product

> TaskFlow Pro — 팀 업무 관리 풀스택 웹 앱

---

## 목표

> **"지금 누가 뭐 해?"** 라는 질문이 사라지게 만든다.

팀 업무를 실시간으로 시각화해, 구두 확인 없이도 누가 무엇을 언제까지 하는지 한눈에 파악할 수 있는 환경을 제공한다.

---

## 페르소나

| 항목 | 내용 |
|------|------|
| **대상** | 10명 규모 스타트업의 팀 리더 |
| **연령대** | 30~40대 |
| **주요 고통** | 진행 상황 파악을 위해 매번 팀원에게 직접 물어봐야 함 |
| **기대 가치** | 슬랙/카카오 대신 한 화면에서 업무 현황 확인 |

---

## MVP 범위

### 포함 (In Scope)

| 기능 | 상세 |
|------|------|
| **CRUD 4종** | 태스크 추가 / 목록 조회 / 수정 / 삭제 — 모두 화면(UI)에서 실행 가능 |
| **상태 분류** | `Todo` / `In Progress` / `Done` 3단계 |
| **마감 시각 (due_at)** | 날짜 + 시간까지 지정 (예: `2026-05-12 18:00`). 날짜만 입력 불가 |
| **테마 토글** | 라이트 / 다크 전환, `localStorage`에 저장 (새로고침 후에도 유지) |
| **모바일 반응형** | 360px 이상 전 해상도에서 레이아웃 깨짐 없음 |

### 제외 (Out of Scope)

| 항목 | 제외 이유 |
|------|----------|
| 외부 결제 | MVP 수익 모델 없음 |
| 네이티브 앱 | 웹 반응형으로 대체 |
| WebRTC | 화상회의 기능 불필요 |
| 외부 캘린더 연동 | Google Calendar 등 연동 미포함 |
| 파일 업로드 | 스토리지 비용·복잡도 증가 |
| WebSocket | MVP는 **폴링**으로 대체. 실시간 동기화는 확장 단계에서 구현 |

## UI 톤 & 스타일

**Mac OS 스타일**을 기준으로 삼는다.

| 요소 | 적용 방식 |
|------|----------|
| **모서리** | 둥근 모서리 (border-radius 8~16px) |
| **그림자** | 부드러운 다층 그림자 (box-shadow, 강한 드롭섀도 금지) |
| **카드** | 반투명 배경 (backdrop-filter blur + 약한 opacity) |
| **폰트** | 시스템 폰트 스택 (`-apple-system, BlinkMacSystemFont, "Segoe UI", sans-serif`) |
| **색상** | 라이트: 흰/회색 계열 / 다크: 짙은 회색 계열. 원색 억제 |

## 확장 로드맵 (MVP 이후)

MVP 완료 후 아래 순서로 확장한다. 현재 코드 작성 시 이 목록을 의식하되, 미리 구현하지 않는다.

1. **JWT 로그인** — 회원가입 · 로그인 · 토큰 갱신
2. **팀(Team) 기능** — 워크스페이스 · 멤버 초대 · 권한 관리
3. **Kanban 보드** — 드래그 앤 드롭 상태 전환
4. **채팅** — 태스크 댓글 · 인앱 알림 (WebSocket 도입)
5. **CI/CD** — GitHub Actions 자동 테스트 · 배포 파이프라인

---

## 성공 기준 (Definition of Done — MVP)

아래 5가지를 모두 만족해야 MVP 완료로 간주한다.

| # | 기준 | 측정 방법 |
|---|------|----------|
| 1 | 새로고침 후 데이터 유지 | DB에 저장된 태스크가 페이지 리로드 후 그대로 표시 |
| 2 | 360px에서 레이아웃 깨짐 없음 | Chrome DevTools 모바일 뷰 육안 확인 |
| 3 | API 응답 200ms 이내 | Network 탭 기준, 로컬 환경 |
| 4 | CRUD 4종 화면에서 동작 | 추가·조회·수정·삭제 모두 버튼/폼 통해 실행 가능 |
| 5 | 테마 토글 작동 | 전환 후 새로고침해도 선택한 테마 유지 |
